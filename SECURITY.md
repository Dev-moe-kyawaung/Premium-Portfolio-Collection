# Security Policy

## Supported Versions

| Version | Status |
|---------|--------|
| V8.x (all themes) | ✅ Actively supported |
| V7.x and below | ❌ Not supported |

## Reporting a Vulnerability

This is a static HTML portfolio website with no server-side code,
databases, or authentication. Security vulnerabilities are unlikely
but still possible (e.g., XSS in contact form simulation).

**To report a security concern:**

1. **Do NOT** open a public GitHub Issue for security vulnerabilities
2. Email directly: **moekyawaung@fastmail.com**
3. Subject line: `[SECURITY] Brief description`
4. Include: description, steps to reproduce, potential impact

**Response time:** Within 48 hours.

## Scope

In scope:
- Cross-site scripting (XSS) in any interactive element
- Content injection vulnerabilities
- Malicious external resource loading

Out of scope:
- Issues with third-party CDNs (Google Fonts, Font Awesome)
- Browser-specific rendering bugs
- Accessibility issues (report as regular issues)
